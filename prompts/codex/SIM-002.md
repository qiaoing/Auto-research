## 功能要求

### 1. `PIDController` 类

在 `src/controllers/pid.py` 中实现：

```python
class PIDController:
    def __init__(
        self,
        kp: float | np.ndarray,
        ki: float | np.ndarray = 0.0,
        kd: float | np.ndarray = 0.0,
        output_limits: tuple[float | np.ndarray | None, float | np.ndarray | None] | None = None,
        integral_limits: tuple[float | np.ndarray | None, float | np.ndarray | None] | None = None,
    ) -> None: ...

    def reset(self) -> None: ...

    def compute(self, error: float | np.ndarray, dt: float) -> float | np.ndarray: ...
```

实现细节：

1. `kp`, `ki`, `kd` 可以是标量或 NumPy array，可通过 NumPy broadcast 与 error 兼容。
2. `compute(error, dt)`：
   - `error` 可以是标量或向量。
   - `dt` 必须为正数；`dt <= 0` 时抛出 `ValueError`。
   - 第一次调用时，微分项 derivative 应使用 0，而不是除以不存在的 previous error。
   - 后续调用：`derivative = (error - previous_error) / dt`。
   - 积分项：`integral += error * dt`。
   - 输出：`kp * error + ki * integral + kd * derivative`。
3. 支持 `output_limits=(lower, upper)`：
   - `lower` 或 `upper` 可为 `None`。
   - 使用 `np.maximum` / `np.minimum` 或 `np.clip` 实现。
   - 标量和向量都应工作。
4. 支持 `integral_limits=(lower, upper)`：
   - 用于 anti-windup，限制 integral state。
   - `None` 表示不限制对应方向。
5. `reset()` 清空积分项和 previous error，使控制器回到初始状态。
6. 对标量输入，`compute` 返回 Python `float` 或 NumPy scalar 均可；但测试中要容易断言。
7. 对向量输入，`compute` 返回 `np.ndarray`。
8. 保留一个向后兼容别名：

```python
PlaceholderPIDController = PIDController
```

这样旧代码如果导入 `PlaceholderPIDController` 不会断。

## 测试要求

创建 `tests/test_pid.py`，至少覆盖以下行为：

1. **比例项测试**：`kp=2, ki=0, kd=0, error=3, dt=0.1` 输出为 `6`。
2. **积分项累积测试**：`kp=0, ki=1, kd=0`，连续两次输入 `error=2, dt=0.5`，第二次输出应体现 integral = 2.0。
3. **微分项测试**：第一次调用 derivative 为 0；第二次 error 从 1 到 3，`dt=0.5`，derivative 为 4。
4. **输出限幅测试**：设置 `output_limits=(-1, 1)`，大误差输出被 clip 到 1 或 -1。
5. **积分限幅测试**：设置 `integral_limits=(-0.5, 0.5)`，长期正误差不会让 integral 超过 0.5。
6. **reset 测试**：调用 reset 后，积分项和 previous error 清零；再次第一次 compute 不含 derivative spike。
7. **向量误差测试**：输入 `np.array([1.0, -2.0, 0.5])`，输出 shape 正确且数值符合预期。
8. **非法 dt 测试**：`dt=0` 和 `dt<0` 抛出 `ValueError`。
